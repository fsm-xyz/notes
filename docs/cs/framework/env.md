# 目标

给业务开发人员提供一个友好，便捷，舒服的环境，提高工程效率

## 历史

业务初期，环境鱼龙混杂

+ 环境没细分，可直连线上调试
+ 本地不能连测试环境调试
+ 生产，测试，开发

### 本地无法连接环境

业务开发人员需要本地搭建开发环境(mysql, redis等)，如果依赖其他服务，还需要启动相关服务

## 模式

### consul/etcd模式

+ 框架需要支持本地服务不注册到qa环境
+ 本地环境可以连上qa环境的服务

### istio模式

+ 本地业务不会注册到qa环境
+ 本地环境可以连上qa环境的服务

## 服务配置

业务部署的时候，可以选择环境，配置，对应的代码版本，部署系统进行动态绑定

不同环境配置文件保持一份即可，如果不兼容，可递增版本
